# VigilAI V2 产品信息架构与页面原型说明

## 1. 文档目标

本文用于定义 VigilAI 在 `广义技术人搞钱机会聚合` 方向下的 V2 产品结构。

目标不是继续把产品做成“信息列表站”，而是把它升级为一个：

> 面向技术人的机会工作台

V2 的核心任务有 4 个：

1. 让用户更快知道今天先看什么
2. 让不同主题的机会内容有主次，不再混乱平铺
3. 让用户可以持续跟进，而不是看完就走
4. 让产品从“聚合内容”升级成“组织行动”

---

## 2. V2 总体定位

### 2.1 一句话定位

VigilAI V2 是一个帮助技术人持续发现、筛选、跟进、订阅高价值机会的一站式机会工作台。

### 2.2 V2 目标用户

主要用户分为 3 类：

- `机会猎手型用户`
  关注 hackathon、grant、bounty、airdrop、quest、testnet 等，追求高频发现和高行动效率
- `广谱技术机会用户`
  希望一站式看到比赛、激励、活动、生态扶持、行业机会
- `重度跟进型用户`
  不只想看内容，还希望收藏、提醒、整理、复盘

### 2.3 V2 的产品原则

- 首页优先展示“今天最值得看什么”，而不是“全部信息”
- 内容要有层级，不同类型机会不能完全同权平铺
- 详情页必须能支持判断与下一步行动
- 工作流优先于资讯流
- 机会发现、机会判断、机会跟进必须形成闭环

---

## 3. 内容分层策略

V2 继续保留当前较宽的内容主题，但做严格分层。

### 3.1 第一层：可直接行动的机会

这是主内容层，默认优先展示。

- hackathon
- grant
- bounty
- data competition
- coding competition
- airdrop
- quest
- testnet

### 3.2 第二层：强相关机会信号

这类内容本身不一定立即可变现，但会影响用户的下一步判断。

- 生态扶持计划
- 开发者招募活动
- 平台任务活动
- 带资源、奖金或生态权益的 dev event

### 3.3 第三层：背景情报

这类内容不再作为主信息流平铺，而是作为背景上下文服务于机会判断。

- 行业新闻
- 平台动态
- 媒体报道
- 社区讨论摘要

### 3.4 内容进入主流的规则

对于第二层和第三层内容，只有满足以下至少一项时，才进入主流推荐：

- 直接关联某个高价值机会
- 对近期判断有明显帮助
- 带有明确行动窗口
- 与用户已关注主题高度相关

---

## 4. V2 信息架构

### 4.1 顶层导航

V2 建议采用以下一级导航：

1. `工作台`
2. `机会池`
3. `情报中心`
4. `日报`
5. `跟进`
6. `来源健康`
7. `设置`

### 4.2 一级导航职责

| 导航 | 主要职责 | 目标用户动作 |
|------|----------|-------------|
| 工作台 | 汇总最值得优先处理的机会与变化 | 先判断今天看什么 |
| 机会池 | 全量机会浏览与筛选 | 查找与探索 |
| 情报中心 | 看最近发生了什么变化 | 理解动态与趋势 |
| 日报 | 自动生成和查看简报 | 快速获取摘要 |
| 跟进 | 收藏、待办、截止提醒 | 管理个人行动 |
| 来源健康 | 看来源质量与异常 | 维护系统可信度 |
| 设置 | 管理关注偏好与提醒方式 | 个性化配置 |

### 4.3 默认首页

默认首页从当前的“活动列表页”切换为“工作台”。

原因：

- V2 的主价值不再是展示所有信息
- 而是告诉用户“今天该先看什么”

---

## 5. 页面原型说明

## 5.1 页面一：工作台

### 页面目标

作为 V2 首页，帮助用户在 30 秒内完成 3 件事：

1. 看懂今天的机会概况
2. 发现最高优先级机会
3. 明确下一步动作

### 页面结构

#### 模块 A：顶部导航区

- Logo / 产品名
- 全局搜索框
- 快捷入口：导入来源、生成日报、查看跟进
- 用户入口

#### 模块 B：今日概览卡片

建议展示 5 张卡片：

- 今日新增机会
- 高价值机会数
- 72 小时内截止机会数
- 今日重要变化数
- 来源异常数

卡片点击后分别进入对应过滤视图。

#### 模块 C：今日重点机会

以列表或卡片形式展示 5 到 10 条机会。

每条卡片显示：

- 标题
- 机会类型
- 来源
- 奖励或价值标签
- 截止时间
- 推荐原因
- 快捷动作：查看详情 / 收藏 / 标记跟进

#### 模块 D：最近 7 天趋势

可视化展示：

- 最近 7 天新增机会数
- 最近 7 天高价值机会数
- 分类趋势

目的不是做 BI，而是帮助用户感知机会热度变化。

#### 模块 E：今日晨报

展示系统自动生成的简版摘要：

- 今日最值得关注的 3 到 5 个主题
- 今日最重要的 3 条机会
- 今日新增异常

并提供：

- 查看完整晨报
- 一键复制摘要
- 发送到邮箱 / 飞书 / Telegram

#### 模块 F：异常与缺口

展示：

- 抓取失败来源
- 连续异常来源
- 某些重点主题今日无更新

这会把“来源异常”转化成真正可见的产品能力，而不是隐藏在系统后台。

#### 模块 G：First Action 行动引导

给用户一个明确动作，例如：

- 完善关注偏好
- 处理 3 条临近截止机会
- 导入自定义来源
- 查看今天的高价值机会

### 页面交互逻辑

- 首次进入时，优先引导用户设置关注主题
- 如果用户没有跟进行为，系统推荐“收藏第一条机会”
- 如果有异常来源，右侧固定显示异常提示
- 如果今日无重点机会，自动切换为“推荐关注的主题”

---

## 5.2 页面二：机会池

### 页面目标

作为全量机会浏览页，承接搜索、筛选、比较、收藏等动作。

### 页面结构

#### 模块 A：顶部筛选栏

建议支持：

- 机会类型
- 来源
- 奖励级别
- 截止时间
- 新旧状态
- 推荐优先级
- 是否已收藏

#### 模块 B：排序工具

排序建议统一为：

- 推荐优先级
- 最新发布
- 最快截止
- 奖励最高
- 来源可信度

#### 模块 C：机会列表

列表卡片建议展示：

- 标题
- 类型标签
- 来源
- 奖励
- 截止时间
- 推荐分
- 机会摘要
- 推荐理由

#### 模块 D：批量操作区

建议支持：

- 批量收藏
- 批量加入跟进
- 批量忽略
- 批量生成摘要

### 页面说明

机会池不是首页，因为它是“探索空间”，不是“优先级入口”。

---

## 5.3 页面三：机会详情

### 页面目标

帮助用户完成“是否值得投入”这一步判断。

### 页面结构

#### 模块 A：主信息区

- 标题
- 类型
- 来源
- 发布时间
- 奖励与权益
- 截止时间
- 官方链接

#### 模块 B：AI 摘要区

建议包含：

- 这是什么机会
- 为什么值得关注
- 适合什么人
- 行动门槛是什么

#### 模块 C：原文与结构化信息区

包括：

- 详细描述
- 组织方
- 地点
- 时间节点
- 标签
- 图片

#### 模块 D：行动区

支持：

- 去原站
- 收藏
- 加入跟进
- 设置提醒
- 加入日报候选

#### 模块 E：相似机会 / 相关机会

展示：

- 同主题机会
- 同平台其他机会
- 相似奖励结构机会

### 页面设计原则

详情页必须比列表页多提供“判断信息”和“下一步动作”，否则它就只是更长的列表卡。

---

## 5.4 页面四：情报中心

### 页面目标

展示“最近发生了什么变化”，而不是只展示“有哪些机会”。

这是 V2 与普通聚合站拉开差距的关键页面。

### 页面结构

#### 模块 A：变化流

每条变化卡片展示：

- 变化对象
- 变化类型
- 变化时间
- 涉及来源
- 变化摘要
- 影响判断

#### 模块 B：变化类型筛选

建议支持：

- 新增
- 更新
- 截止临近
- 奖励变化
- 规则变化
- 来源异常恢复

#### 模块 C：主题聚合视图

支持按主题查看变化，例如：

- Web3 激励
- Hackathon
- Grant
- Bounty
- 技术竞赛

### 页面价值

这个页面解决的是“我不想重刷整个列表，我只想知道最近有什么值得注意的变化”。

---

## 5.5 页面五：日报

### 页面目标

把分散内容转化为“可阅读、可转发、可复盘”的日报产品。

### 页面结构

#### 模块 A：日报列表

展示：

- 今日晨报
- 昨日日报
- 周报
- 专题简报

#### 模块 B：日报详情

建议结构：

- 今日总览
- 今日重点机会
- 今日变化摘要
- 今日异常来源
- 编辑备注

#### 模块 C：发送与导出

支持：

- 导出 Markdown
- 导出邮件版
- 发送到飞书 / Telegram / 邮箱

### 日报的定位

日报不是附属功能，而是 V2 的核心交付形式之一。  
它能显著提高用户复访与团队传播效率。

---

## 5.6 页面六：跟进

### 页面目标

把“看过”升级成“持续行动”。

### 页面结构

#### 模块 A：我的收藏

展示用户手动收藏的机会。

#### 模块 B：待跟进

支持状态：

- 待判断
- 已关注
- 准备参与
- 已提交
- 已忽略

#### 模块 C：提醒中心

包括：

- 截止提醒
- 关注主题新增提醒
- 收藏机会状态变化提醒

### 页面价值

这个页面是机会产品形成留存的关键。  
没有跟进页，用户行为会永远停在“浏览”。

---

## 5.7 页面七：来源健康

### 页面目标

让来源体系更透明、更可信。

### 页面结构

#### 模块 A：来源健康总览

展示：

- 总来源数
- 正常来源数
- 异常来源数
- 今日无更新来源数
- 今日恢复来源数

#### 模块 B：来源列表

每个来源展示：

- 来源名
- 类型
- 最近成功时间
- 最近有效更新数
- 成功率
- 健康度
- 当前状态

#### 模块 C：异常面板

展示：

- 连续失败来源
- 空结果来源
- 结构变化来源
- 需要人工处理来源

### 页面定位

这是系统可信度页面。  
它主要面向重度用户和维护者，但也能提升普通用户对平台结果的信任。

---

## 5.8 页面八：设置

### 页面目标

承接个性化和订阅配置。

### 核心设置项

- 关注的机会类型
- 关注的技术主题
- 不想看的来源
- 提醒方式
- 日报频率
- 时区与语言

---

## 6. 核心交互闭环

V2 应该形成下面这条闭环：

```mermaid
flowchart LR
    A[工作台发现重点机会] --> B[查看详情并判断]
    B --> C[收藏或加入跟进]
    C --> D[设置提醒或加入日报]
    D --> E[临期提醒或状态更新]
    E --> F[回到跟进页处理]
```

如果没有这条闭环，产品仍然只是“内容浏览器”。

---

## 7. 机会评分与优先级机制

V2 建议引入统一评分。

### 7.1 评分维度

- `价值分`
  奖励金额、生态价值、资源价值
- `时效分`
  是否新发布、截止时间是否临近
- `适配分`
  是否匹配用户偏好
- `可信分`
  来源可靠性、信息完整度、多来源交叉验证情况

### 7.2 评分用途

统一用于：

- 工作台重点推荐
- 机会池默认排序
- 日报候选排序
- 提醒优先级

### 7.3 产品原则

评分不是黑盒推荐，而是要给出可解释理由。

例如：

- 高价值：奖金高、官方生态扶持、截止近
- 高优先级：你关注的主题、最近 24 小时新增、来源可信

---

## 8. V2 与当前版本的最大差异

| 当前版本 | V2 |
|---------|----|
| 首页是活动列表 | 首页是机会工作台 |
| 内容平铺展示 | 内容按优先级和主题分层 |
| 用户主要做浏览 | 用户要完成发现、判断、跟进 |
| 详情页偏信息展示 | 详情页偏判断与行动 |
| 日报是可选能力 | 日报是核心交付形式 |
| 来源页偏运维 | 来源健康成为可信度模块 |

---

## 9. V2 最小可上线范围

如果只做一个最小版本，建议优先上线：

1. 工作台首页
2. 机会池重构
3. 详情页增强
4. 跟进页
5. 基础日报

来源健康和情报中心可以作为第二批上线。

原因：

- 这 5 个模块已经足够形成用户闭环
- 能先验证“工作台”是否优于“纯列表”

---

## 10. 版本建议

### V2.0

聚焦用户价值闭环：

- 工作台
- 机会池
- 详情页
- 跟进页
- 基础日报

### V2.1

聚焦情报能力：

- 情报中心
- 变化聚合
- 智能去重
- 来源可信度

### V2.2

聚焦分发与订阅：

- 多渠道提醒
- 自定义日报
- 团队共享
- Webhook

---

## 11. 一句话总结

V2 的本质不是把 VigilAI 做成“更大的信息站”，而是把它做成一个：

> 帮技术人持续发现、判断、跟进机会的一站式机会工作台

如果 V1 解决的是“我能不能看到机会”，那么 V2 要解决的是：

> “我今天应该先看哪几个机会，以及接下来该做什么。”
