# VigilAI V2 核心页面组件清单

## 1. 文档目标

本文用于把 V2 的核心页面进一步拆解为组件级清单，方便后续：

- 产品继续细化交互
- 设计制作高保真稿
- 前后端拆开发任务
- 测试明确页面验收范围

本次只聚焦 V2 最关键的三个页面：

1. 工作台首页
2. 机会池
3. 机会详情页

---

## 2. 组件拆解原则

组件拆解遵循 4 个原则：

- `高内聚`
  一个组件只负责一类清晰任务
- `可复用`
  通用展示单元尽量可在多个页面复用
- `数据边界清晰`
  明确哪些组件需要直接拿数据，哪些只接收 props
- `优先级明确`
  区分必须优先开发的组件和可后置优化组件

## 3. 页面一：工作台首页

## 3.1 页面目标

帮助用户在最短时间完成三件事：

1. 感知今天的机会概况
2. 看到今天最值得先看的机会
3. 明确接下来应该做什么

## 3.2 页面组件树

```text
WorkspacePage
├─ WorkspaceHeader
├─ GlobalSearchBar
├─ QuickActionBar
├─ OverviewMetricsGrid
│  ├─ MetricCard
│  ├─ MetricCard
│  ├─ MetricCard
│  ├─ MetricCard
│  └─ MetricCard
├─ PriorityOpportunityPanel
│  ├─ PriorityOpportunityList
│  │  └─ PriorityOpportunityItem
├─ DailyDigestPanel
├─ TrendChartPanel
├─ AlertSourcePanel
├─ FirstActionPanel
└─ RecommendedTopicsPanel
```

## 3.3 组件清单

### 组件 1：`WorkspacePage`

职责：

- 作为首页容器组织各模块布局
- 统一拉取首页所需聚合数据

依赖数据：

- 今日概览数据
- 高优先级机会列表
- 趋势数据
- 日报摘要
- 异常来源

关键状态：

- `loading`
- `error`
- `refreshing`

优先级：

- `P0`

### 组件 2：`WorkspaceHeader`

职责：

- 展示页面标题、欢迎语、今日扫描概览

依赖数据：

- 今日日期
- 今日扫描结果概览

关键交互：

- 手动刷新

优先级：

- `P0`

### 组件 3：`GlobalSearchBar`

职责：

- 支持全局搜索机会、来源、日报

依赖数据：

- 搜索输入
- 搜索建议

关键交互：

- 输入关键词
- 回车搜索
- 点击建议跳转

优先级：

- `P1`

### 组件 4：`QuickActionBar`

职责：

- 提供高频快捷操作入口

建议动作：

- 导入来源
- 查看跟进
- 生成日报
- 查看高价值机会

优先级：

- `P0`

### 组件 5：`OverviewMetricsGrid`

职责：

- 聚合展示首页 5 张核心指标卡

子组件：

- `MetricCard`

建议指标：

- 今日新增机会
- 高价值机会数
- 72 小时内截止
- 今日重要变化
- 异常来源数

关键交互：

- 点击卡片跳转到对应过滤视图

优先级：

- `P0`

### 组件 6：`MetricCard`

职责：

- 展示单项指标的数值、标签、变化说明

字段建议：

- 标题
- 数值
- 辅助说明
- 趋势标签

优先级：

- `P0`

### 组件 7：`PriorityOpportunityPanel`

职责：

- 承载首页最重要的机会列表

依赖数据：

- 首页推荐机会列表

关键逻辑：

- 默认只展示前 5 到 10 条
- 可切换“高价值 / 临近截止 / 最新发布”

优先级：

- `P0`

### 组件 8：`PriorityOpportunityList`

职责：

- 负责渲染重点机会列表

子组件：

- `PriorityOpportunityItem`

优先级：

- `P0`

### 组件 9：`PriorityOpportunityItem`

职责：

- 展示一条重点机会的关键字段和操作入口

字段建议：

- 标题
- 类型
- 来源
- 奖励
- 截止时间
- 推荐理由

关键交互：

- 查看详情
- 收藏
- 加入跟进

优先级：

- `P0`

### 组件 10：`DailyDigestPanel`

职责：

- 展示今日晨报的简版摘要

字段建议：

- 今日总览一句话
- 3 条重点机会摘要
- 1 条异常提醒

关键交互：

- 查看完整日报
- 一键复制摘要

优先级：

- `P0`

### 组件 11：`TrendChartPanel`

职责：

- 展示最近 7 天更新趋势

依赖数据：

- 7 天新增数
- 7 天高价值机会数

优先级：

- `P1`

### 组件 12：`AlertSourcePanel`

职责：

- 展示当前最需要注意的异常来源

依赖数据：

- 异常来源列表

关键交互：

- 查看来源详情
- 手动刷新

优先级：

- `P1`

### 组件 13：`FirstActionPanel`

职责：

- 给用户一个明确的下一步行动建议

建议内容：

- 完善偏好
- 处理临期机会
- 查看高价值机会
- 导入自定义来源

优先级：

- `P0`

### 组件 14：`RecommendedTopicsPanel`

职责：

- 给新用户或低活跃用户推荐值得关注的主题

适用场景：

- 首页无足够个性化数据时兜底

优先级：

- `P2`

## 3.4 首页数据接口建议

建议整合为一个首页聚合接口，而不是前端分别请求多个接口。

建议结构：

```json
{
  "overview": {},
  "priority_items": [],
  "digest_preview": {},
  "trend_7d": [],
  "alerts": [],
  "first_action": {},
  "recommended_topics": []
}
```

## 4. 页面二：机会池

## 4.1 页面目标

承接用户从首页进入后的深入浏览、筛选、比较和批量处理。

## 4.2 页面组件树

```text
OpportunityHubPage
├─ OpportunityFilterBar
├─ OpportunitySortBar
├─ OpportunityBatchActionBar
├─ OpportunityList
│  └─ OpportunityCard
├─ OpportunityPagination
└─ OpportunityPreviewDrawer
```

## 4.3 组件清单

### 组件 1：`OpportunityHubPage`

职责：

- 作为机会池页面容器
- 管理筛选、排序、分页、预览抽屉状态

关键状态：

- `filters`
- `sort`
- `selectedIds`
- `previewId`

优先级：

- `P0`

### 组件 2：`OpportunityFilterBar`

职责：

- 提供多维筛选能力

建议筛选项：

- 类型
- 来源
- 奖励级别
- 截止时间
- 推荐优先级
- 是否已收藏

关键交互：

- 多条件筛选
- 清空筛选

优先级：

- `P0`

### 组件 3：`OpportunitySortBar`

职责：

- 管理排序逻辑

建议排序：

- 推荐优先级
- 最新发布
- 最快截止
- 奖励最高
- 来源可信度

优先级：

- `P0`

### 组件 4：`OpportunityBatchActionBar`

职责：

- 承接多选后的批量操作

建议动作：

- 批量收藏
- 批量加入跟进
- 批量忽略
- 批量生成摘要

显示条件：

- 只有在选中机会时显示

优先级：

- `P1`

### 组件 5：`OpportunityList`

职责：

- 渲染机会列表

子组件：

- `OpportunityCard`

优先级：

- `P0`

### 组件 6：`OpportunityCard`

职责：

- 展示一条机会的核心信息

字段建议：

- 标题
- 类型标签
- 来源
- 奖励
- 截止时间
- 推荐分
- 一句话摘要
- 推荐理由

关键交互：

- 查看详情
- 打开预览
- 收藏
- 加入跟进
- 多选

优先级：

- `P0`

### 组件 7：`OpportunityPagination`

职责：

- 管理分页与页码切换

优先级：

- `P0`

### 组件 8：`OpportunityPreviewDrawer`

职责：

- 右侧快速预览当前选中机会，减少频繁跳页

建议展示：

- 标题
- 摘要
- 奖励
- 时间
- 推荐理由
- 快捷操作

优先级：

- `P1`

## 4.4 机会池数据接口建议

建议保留列表接口，但补强以下字段：

- `score`
- `score_reason`
- `deadline_level`
- `trust_level`
- `is_favorited`
- `is_tracking`

## 5. 页面三：机会详情页

## 5.1 页面目标

把机会从“看见”转化为“判断”和“行动”。

## 5.2 页面组件树

```text
OpportunityDetailPage
├─ OpportunityDetailHeader
├─ OpportunityMetaSection
├─ OpportunitySummarySection
├─ OpportunityReasonPanel
├─ OpportunityTimelineSection
├─ OpportunityContentSection
├─ OpportunityActionPanel
└─ RelatedOpportunitySection
```

## 5.3 组件清单

### 组件 1：`OpportunityDetailPage`

职责：

- 作为详情页总容器
- 统一拉取详情与相关机会数据

关键状态：

- `loading`
- `error`
- `activity`
- `relatedItems`

优先级：

- `P0`

### 组件 2：`OpportunityDetailHeader`

职责：

- 展示标题、类型、来源、发布时间与返回动作

关键交互：

- 返回列表
- 分享链接

优先级：

- `P0`

### 组件 3：`OpportunityMetaSection`

职责：

- 展示最核心的结构化元信息

字段建议：

- 奖励
- 来源
- 组织方
- 地点
- 标签

优先级：

- `P0`

### 组件 4：`OpportunitySummarySection`

职责：

- 展示 AI 摘要和一句话概括

建议结构：

- 这是什么机会
- 适合谁
- 核心收益

优先级：

- `P0`

### 组件 5：`OpportunityReasonPanel`

职责：

- 可解释地告诉用户为什么值得看

内容建议：

- 高价值原因
- 时效原因
- 匹配原因
- 来源可信原因

优先级：

- `P0`

### 组件 6：`OpportunityTimelineSection`

职责：

- 展示机会关键时间线

字段建议：

- 开始时间
- 结束时间
- 截止时间
- 距截止还有几天

优先级：

- `P0`

### 组件 7：`OpportunityContentSection`

职责：

- 展示完整描述与补充信息

内容建议：

- 原始描述
- full_content
- 图片
- 规则说明

优先级：

- `P0`

### 组件 8：`OpportunityActionPanel`

职责：

- 聚合所有下一步行动入口

建议动作：

- 去原站
- 收藏
- 加入跟进
- 设置提醒
- 加入日报候选

要求：

- 固定在右侧或页面下方，始终易达

优先级：

- `P0`

### 组件 9：`RelatedOpportunitySection`

职责：

- 展示相似机会或同主题机会

建议逻辑：

- 同主题
- 同来源
- 同奖励量级

优先级：

- `P1`

## 5.4 详情页数据接口建议

建议详情接口补充以下字段：

- `summary`
- `score`
- `score_reason`
- `timeline`
- `related_items`
- `tracking_status`

## 8. 组件验收关键点

### 首页组件验收

- 是否一进首页就能看到重点机会
- 是否能明确读出“今天先看什么”
- 是否有明确行动引导

### 机会池组件验收

- 是否能快速筛到用户关心的机会
- 是否能用推荐优先级排序
- 是否支持收藏与跟进

### 详情页组件验收

- 是否比列表多提供“判断信息”
- 是否有清晰可解释的推荐理由
- 是否始终能快速进入下一步行动

---

## 9. 一句话总结

这份组件清单的核心目的不是把页面切得更碎，而是把 V2 最关键的用户路径拆清楚：

> 首页负责发现重点，机会池负责深度探索，详情页负责支持判断和行动。

如果后续组件开发始终围绕这条路径展开，V2 才能从“列表站”真正升级成“机会工作台”。
