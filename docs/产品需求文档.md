# VigilAI 产品需求文档

## 1. 产品概述

### 1.1 产品名称
VigilAI - 开发者搞钱机会监控系统

### 1.2 产品定位
一个面向开发者和技术人员的自动化信息监控平台，实时追踪全球范围内的黑客松、编程竞赛、Web3空投、漏洞赏金、开源悬赏、创业资助等变现机会。

### 1.3 核心价值
- 信息差变现：比别人更早发现高价值机会
- 自动化监控：7x24小时不间断追踪，节省人工盯盘时间
- 智能筛选：过滤低价值信息，聚焦高回报机会
- 一站式聚合：整合50+信息源，统一展示和管理

### 1.4 目标用户
- 独立开发者
- 数据科学家/AI工程师
- Web3开发者/安全研究员
- 创业者/自由职业者
- 学生开发者

## 3. 功能需求

### 3.1 数据采集模块

#### 3.1.1 RSS订阅器
- 支持标准RSS/Atom格式解析
- 自动检测更新频率
- 支持自定义订阅源添加
- 目标源：Devpost、36氪、虎嗅等

#### 3.1.2 网页爬虫
- 静态页面爬取（BeautifulSoup）
- 动态页面爬取（Selenium/Playwright）
- 反爬虫策略：User-Agent轮换、请求间隔、代理池
- 增量爬取：只抓取新增内容
- 目标源：DoraHacks、Gitcoin、华为开发者等

#### 3.1.3 API调用器
- Kaggle API集成
- GraphQL查询支持（Gitcoin）
- RESTful API调用
- API密钥管理

### 3.2 数据处理模块

#### 3.2.1 数据清洗
- HTML标签清理
- 文本规范化
- 日期格式统一
- 金额单位转换（USD/CNY/ETH等）

#### 3.2.2 智能去重
- 基于标题相似度去重
- 基于URL去重
- 跨平台同一活动识别

#### 3.2.3 自动分类
- 活动类型分类：黑客松/竞赛/空投/赏金/资助/活动
- 技术领域分类：AI/Web3/移动开发/安全/数据科学
- 奖金等级分类：高(>10万)/中(1-10万)/低(<1万)

#### 3.2.4 关键词过滤
- 自定义关键词白名单/黑名单
- 正则表达式匹配
- 多语言支持（中/英）

### 3.3 数据存储模块

#### 3.3.1 第一阶段（JSON文件）
- activities.json：活动数据
- sources_status.json：信息源状态
- user_preferences.json：用户偏好

#### 3.3.2 第二阶段（数据库）
- PostgreSQL/MongoDB
- 历史数据归档
- 全文搜索索引

### 3.4 定时调度模块

#### 3.4.1 调度策略
- 高优先级：每小时执行
- 中优先级：每2小时执行
- 低优先级：每6小时执行
- 支持手动触发刷新

#### 3.4.2 任务管理
- 任务队列
- 失败重试（最多3次）
- 错误日志记录
- 运行状态监控

### 3.5 API服务模块

#### 3.5.1 活动接口
- GET /api/activities - 获取活动列表（支持分页、筛选、排序）
- GET /api/activities/{id} - 获取活动详情
- POST /api/activities/search - 搜索活动

#### 3.5.2 信息源接口
- GET /api/sources - 获取所有信息源状态
- POST /api/sources/{id}/refresh - 手动刷新指定信息源
- POST /api/sources/refresh-all - 刷新所有信息源

#### 3.5.3 统计接口
- GET /api/stats - 获取统计数据（总数、分类统计、趋势）
- GET /api/stats/daily - 每日新增统计

#### 3.5.4 用户偏好接口
- GET /api/preferences - 获取用户偏好
- PUT /api/preferences - 更新用户偏好

### 3.6 通知推送模块（第二阶段）

#### 3.6.1 推送渠道
- 邮件通知
- 微信/钉钉机器人
- Telegram Bot
- 浏览器推送

#### 3.6.2 推送规则
- 高价值活动即时推送（奖金>10万）
- 每日摘要推送
- 自定义关键词触发推送

### 3.7 前端展示模块

#### 3.7.1 首页仪表盘
- 统计卡片：总活动数、今日新增、各类型数量
- 信息源状态概览
- 最新活动列表

#### 3.7.2 活动列表页
- 卡片式展示
- 多维度筛选：类型、来源、奖金范围、截止时间
- 关键词搜索
- 排序：时间、奖金、热度

#### 3.7.3 活动详情页
- 完整描述
- 奖金信息
- 时间线（报名/提交/评审）
- 外链跳转

#### 3.7.4 信息源管理页
- 各信息源运行状态
- 最后更新时间
- 错误信息展示
- 手动刷新按钮

#### 3.7.5 设置页
- 关键词配置
- 通知偏好
- 显示偏好

## 4. 数据模型

### 4.1 活动模型（Activity）

```
{
  "id": "string",              // 唯一标识
  "title": "string",           // 活动标题
  "description": "string",     // 活动描述
  "source_id": "string",       // 信息源ID
  "source_name": "string",     // 信息源名称
  "url": "string",             // 原始链接
  "category": "string",        // 类型：hackathon/competition/airdrop/bounty/grant/event
  "tags": ["string"],          // 标签：AI/Web3/Mobile等
  "prize": {
    "amount": "number",        // 奖金金额
    "currency": "string",      // 货币单位
    "description": "string"    // 奖金描述
  },
  "dates": {
    "start_date": "datetime",  // 开始时间
    "end_date": "datetime",    // 结束时间
    "deadline": "datetime"     // 报名截止
  },
  "location": "string",        // 地点：online/城市名
  "organizer": "string",       // 主办方
  "status": "string",          // 状态：upcoming/ongoing/ended
  "created_at": "datetime",    // 创建时间
  "updated_at": "datetime"     // 更新时间
}
```

### 4.2 信息源模型（Source）

```
{
  "id": "string",              // 信息源ID
  "name": "string",            // 信息源名称
  "type": "string",            // 类型：rss/web/api
  "url": "string",             // 信息源URL
  "priority": "string",        // 优先级：high/medium/low
  "update_interval": "number", // 更新间隔（秒）
  "enabled": "boolean",        // 是否启用
  "last_run": "datetime",      // 最后运行时间
  "last_success": "datetime",  // 最后成功时间
  "status": "string",          // 状态：running/success/error/idle
  "error_message": "string",   // 错误信息
  "activity_count": "number"   // 已抓取活动数
}
```

### 4.3 用户偏好模型（Preferences）

```
{
  "keywords": {
    "include": ["string"],     // 包含关键词
    "exclude": ["string"]      // 排除关键词
  },
  "categories": ["string"],    // 关注的类型
  "min_prize": "number",       // 最低奖金筛选
  "sources": ["string"],       // 关注的信息源
  "notifications": {
    "email": "boolean",
    "wechat": "boolean",
    "telegram": "boolean"
  }
}
```

## 5. 技术架构

### 5.1 系统架构图

```
┌─────────────────────────────────────────────────────────┐
│                     前端 Web 界面                        │
│              (React + TypeScript + Shadcn-ui)           │
└────────────────────────┬────────────────────────────────┘
                         │ REST API
┌────────────────────────┴────────────────────────────────┐
│                    后端 API 服务                         │
│                 (FastAPI + Python 3.9+)                 │
├─────────────────────────────────────────────────────────┤
│                   定时任务调度器                          │
│                   (APScheduler)                         │
├─────────────────────────────────────────────────────────┤
│                     爬虫系统                             │
│  ┌──────────┬──────────┬──────────┬──────────┐         │
│  │   RSS    │   网页   │   API    │  Web3    │         │
│  │  爬虫    │   爬虫   │  调用    │  爬虫    │         │
│  └──────────┴──────────┴──────────┴──────────┘         │
├─────────────────────────────────────────────────────────┤
│                   数据管理层                             │
│          (去重、分类、清洗、增量更新)                     │
├─────────────────────────────────────────────────────────┤
│                   数据存储层                             │
│         (JSON文件 → PostgreSQL/MongoDB)                 │
└─────────────────────────────────────────────────────────┘
```

### 5.2 技术栈

#### 后端
- 语言：Python 3.9+
- Web框架：FastAPI
- 定时任务：APScheduler
- 网页爬取：BeautifulSoup4、Selenium、Playwright
- RSS解析：feedparser
- HTTP客户端：httpx、aiohttp
- 数据验证：Pydantic

#### 前端
- 框架：React 18
- 语言：TypeScript
- UI组件：Shadcn-ui
- 状态管理：React Query
- 构建工具：Vite

#### 存储
- 第一阶段：JSON文件
- 第二阶段：PostgreSQL + Redis缓存

#### 部署
- 容器化：Docker
- 反向代理：Nginx
