# VigilAI V2 低保真线框说明

## 1. 文档目的

本文用于把 `V2产品信息架构与页面原型说明` 进一步落成可讨论的低保真页面骨架。

目标不是定义视觉风格，而是明确：

- 每个页面要承接什么任务
- 页面信息应该如何分区
- 用户进入页面后第一眼应该看到什么
- 页面里哪些区域是核心，哪些区域是辅助

---

## 2. 全局布局原则

### 2.1 全局结构

V2 统一采用三段式结构：

1. 左侧导航区
2. 主内容区
3. 右侧辅助区

适用页面：

- 工作台
- 情报中心
- 日报
- 来源健康

对于信息密度更高的页面，可以采用：

- 顶部筛选区
- 主列表区
- 右侧详情抽屉

适用页面：

- 机会池
- 跟进

### 2.2 左侧导航统一项

- 工作台
- 机会池
- 情报中心
- 日报
- 跟进
- 来源健康
- 设置

### 2.3 顶部全局区统一项

- 全局搜索
- 导入来源
- 生成日报
- 提醒入口
- 用户入口

### 2.4 右侧辅助区统一承载内容

- 今日晨报摘要
- 异常来源提醒
- First Action
- 快捷推荐

---

## 3. 页面一：工作台

### 3.1 页面目标

让用户 30 秒内知道：

- 今天新增了什么
- 最值得先看哪几条
- 哪些事情需要立即处理

### 3.2 低保真结构

```text
+----------------------------------------------------------------------------------+
| 左侧导航          | 顶部：标题 / 搜索 / 导入来源 / 生成日报 / 用户入口            |
|------------------|---------------------------------------------------------------|
| 工作台            | [今日概览卡1] [卡2] [卡3] [卡4] [卡5]                         |
| 机会池            |                                                               |
| 情报中心          | [最近7天趋势图]                         [今日晨报摘要]         |
| 日报              |                                                               |
| 跟进              | [今日重点机会列表]                        [异常来源]           |
| 来源健康          |                                                               |
| 设置              | [First Action 行动引导]                   [快捷推荐]           |
+----------------------------------------------------------------------------------+
```

### 3.3 首屏优先级

首屏从上到下的优先级必须固定：

1. 今日概览
2. 今日重点机会
3. 今日晨报
4. 异常来源
5. 趋势图
6. 行动引导

### 3.4 工作台卡片清单

建议固定 5 张概览卡：

- 今日新增机会
- 高价值机会
- 72 小时内截止
- 今日重要变化
- 异常来源

### 3.5 今日重点机会模块结构

每条卡片建议包含：

- 标题
- 类型标签
- 奖励信息
- 截止时间
- 推荐原因
- 操作按钮：查看 / 收藏 / 跟进

## 4. 页面二：机会池

### 4.1 页面目标

作为全量浏览入口，支持筛选、比较、收藏、批量处理。

### 4.2 低保真结构

```text
+----------------------------------------------------------------------------------+
| 顶部搜索 + 筛选栏                                                                |
| 类型 | 来源 | 奖励级别 | 截止时间 | 推荐优先级 | 是否已收藏 | 排序               |
|----------------------------------------------------------------------------------|
| 机会列表区                                                  | 右侧详情抽屉       |
|------------------------------------------------------------|--------------------|
| [机会卡片1]                                                 | 默认显示所选机会   |
| [机会卡片2]                                                 | 摘要 / 奖励 / 时间 |
| [机会卡片3]                                                 | 推荐原因 / 动作    |
| [机会卡片4]                                                 |                    |
| 分页 / 批量操作                                              |                    |
+----------------------------------------------------------------------------------+
```

### 4.3 机会卡片骨架

```text
[类型] [来源]                          [推荐分]
标题
一句话摘要
奖励信息 | 截止时间 | 标签
[查看详情] [收藏] [加入跟进]
```

### 4.4 批量操作区

建议固定在底部或列表上方：

- 批量收藏
- 批量加入跟进
- 批量忽略
- 批量生成摘要

---

## 5. 页面三：机会详情

### 5.1 页面目标

帮助用户完成“值不值得做”和“接下来怎么做”的判断。

### 5.2 低保真结构

```text
+----------------------------------------------------------------------------------+
| 返回按钮 / 标题 / 类型标签 / 来源 / 发布时间                                      |
|----------------------------------------------------------------------------------|
| 左侧主信息区                                              | 右侧行动区          |
|-----------------------------------------------------------|---------------------|
| 奖励与权益                                                | [去原站]            |
| 截止时间                                                  | [收藏]              |
| AI摘要：这是什么、为什么值得关注、适合谁                   | [加入跟进]          |
| 详细描述                                                  | [设置提醒]          |
| 组织方 / 地点 / 标签 / 图片                                | [加入日报候选]      |
| 相似机会 / 相关机会                                       |                     |
+----------------------------------------------------------------------------------+
```

### 5.3 页面关键要求

- 行动区始终可见
- 推荐理由必须露出
- 截止时间必须高亮
- 相关机会不能埋太深

---

|-----------------------------|-----------------------------------|--------------------|

| 昨日日报                     | 今日重点机会                      | 邮件发送           |
| 周报                         | 今日变化摘要                      | 飞书发送           |
| 专题简报                     | 今日异常来源                      | Telegram 发送      |

### 7.3 日报详情结构

- 今日总览
- 今日重点机会
- 今日重要变化
- 今日异常与缺口
- 编辑备注

## 9. 页面七：来源健康

### 9.1 页面目标

提升平台的结果可信度，并给重度用户或维护者可见的来源状态。

### 9.2 低保真结构

```text
+----------------------------------------------------------------------------------+
| 健康概览卡：总来源 / 正常 / 异常 / 今日无更新 / 今日恢复                         |
|----------------------------------------------------------------------------------|
| 来源列表区                                                  | 异常面板            |
|------------------------------------------------------------|---------------------|
| 来源A  健康度  成功率  最近更新时间  最近有效更新           | 连续失败来源        |
| 来源B  健康度  成功率  最近更新时间  最近有效更新           | 空结果来源          |
| 来源C  健康度  成功率  最近更新时间  最近有效更新           | 结构变化来源        |
+----------------------------------------------------------------------------------+
```

### 9.3 行内操作

每个来源建议支持：

- 查看详情
- 手动刷新
- 暂停
- 调整优先级
- 查看历史异常

## 11. 移动端适配原则

移动端不建议完全复制桌面工作台结构，建议采用：

- 首页工作台简化版
- 今日重点机会优先
- 晨报卡片前置
- 异常来源折叠
- 跟进与提醒单独强化

移动端最核心的不是看全量，而是：

- 快速看重点
- 快速收藏
- 快速加入跟进

---

## 12. 线框评审时的关键问题

这份低保真线框在评审时，需要重点验证以下问题：

1. 首页是否真的比列表页更能帮助用户进入状态
2. 重点机会模块是否足够明确地体现优先级
3. 跟进页是否能形成留存闭环
4. 日报页是否值得成为核心模块，而不是附属功能
5. 来源健康是否对普通用户也有感知价值

---

## 13. 一句话总结

V2 的页面结构不应该继续围绕“展示所有机会”设计，而应该围绕：

> 让用户更快发现重点、形成判断、进入跟进

因此所有线框的核心原则只有一句话：

> 首页做工作台，列表做探索，详情做判断，跟进做留存，日报做分发，来源健康做可信度。
